❓ FAQ — Quick Reference
What is run_time?
When the model was computed. HRRR updates every hour, ECMWF twice daily, NBM/GFS-MOS several times per day. This tool always uses the latest run for every model.
What is valid_time?
The specific hour being predicted. The tool finds the hour within your window with the highest temperature and shows that as the Daily High.

What is the Quick Signal?
SignalCondition
🟢 Stable SetupSpread ≤ 2°F AND Score B ≥ 2
🟡 Medium RiskSpread 3–4°F OR Score B = 1

What is Consensus High?
Average of all 3 model highs. Use this as your primary input for the Temperature Trader tool.
What is Evolution Score B?
SpreadStabilityScore B
0–1°FStable ✅3
2°FMedium ⚠️2
3°FRisky ⚠️1
4°F+Unstable ❌0
What is the Live WU High?
The current observed trading-day high using WU logic — exactly what Polymarket uses to resolve the market. Check intraday to see where resolution is tracking.
Why WU Logic?
Polymarket resolves via Weather Underground using METAR observations and Local Civil Time. This tool fetches with logic=wu to match exactly.
Is my API key stored?
Only in sessionStorage. Survives a refresh but clears when you close the tab. Never written to any server or file.
Is DST handled automatically?
Yes. JavaScript's Intl API handles DST for all cities. No manual changes needed on March 8 or November 1.

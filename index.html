❓ FAQ — Quick Reference

Everything you need to remember about this tool and the trading workflow.

What is run_time?
The time the model was computed. Not a temperature. Think of it as the "printed on" date of the forecast. HRRR updates every hour, ECMWF twice daily.
What is valid_time?
The specific hour being predicted. The temperature_f in that row is the expected temperature AT that hour — not the daily high.
Why use only the latest HRRR run?
HRRR runs every hour. Older runs are stale. The latest run = the most accurate current forecast. This tool filters to latest run automatically.
Why show 2 ECMWF runs?
ECMWF runs twice daily. Comparing Run 1 vs Run 2 tells you if the forecast shifted between runs — this is your evolution signal. A 10°F shift = very unstable = likely SKIP.
How is the Daily High calculated?
The API returns hourly rows. This tool takes the maximum temperature_f across all rows within the time window.

What is WU Logic? Why does it matter?
Polymarket resolves temperature markets using Weather Underground (WU) data. WU uses METAR observations only and Local Civil Time. This tool fetches the live WU high using logic=wu so you see exactly what Polymarket is tracking.
What is the Live WU High?
The current trading-day high temperature as calculated by Wethr using WU logic. This is the number Polymarket will use to resolve the market. Check this intraday to see where the market is heading.
Why are times shown in UTC AND local?
The API always returns UTC. Local time is shown so you can quickly understand when the forecast peak is expected without doing the math yourself.
Is DST handled automatically?
Yes. JavaScript's Intl API handles DST automatically for all 3 cities. You never need to adjust times manually — not on March 8 or November 1.
What does "Today" mean if I'm in a different timezone?
"Today" always means the current date in the target city — not your local date. The city clocks at the top show you exactly what date each city is on right now.
Is my API key stored anywhere?
No. It lives only in your browser while the page is open. It is never written to the HTML file, never sent to GitHub, and cleared the moment you close the tab. Paste it fresh each session.
What if I get a CORS error?
CORS is a browser security rule that may block API calls from web pages. If you see a CORS error, the results section will show a clear explanation and next steps. The fix is to set up a free Cloudflare Workers proxy.
